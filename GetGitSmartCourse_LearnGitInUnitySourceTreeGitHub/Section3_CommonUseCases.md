# Section 3: Common Uses Cases

### Intro & Stashing
+ Stashing gets you back to where you last committed while saving unexpected changes

### Commit
+ each one has a unique SHA cryptographic number
+ save changes in your local repo ready to push to the remote repo

### How to use Branches
+ Branch is a pointer to a commit
+ Branch lets you commit changes from the master branch

### Introducing Merging
+ merges two different branches
+ merge early, merge often

### Simple Merge Conflicts
+ solve simple merge conflicts by choosing whether to use the current or the other branch's change

### Revert or Reverse Commit
+ revert lets you revert changes done
+ reverse lets you go back to a previous commit
